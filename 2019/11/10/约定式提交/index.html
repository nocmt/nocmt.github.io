约定式提交

Words count 641 Reading time 1 mins. Read count 0

git commit -m “” 的时候总是不知道怎么填，版本更新日志混乱无序？

试试「约定式提交」

fix: 在代码库中修复了一个bug（这和语义化版本中的PATCH相对应）
feat: 在代码库中新增了一个功能（这和语义化版本中的MINOR相对应）
build: 修改了与构建相关的配置。比如npm脚本、DockerFile、MakeFile等
docs: 修改了文档
test: 增加测试内容
chore: 日常更新
refactor: 没有修复Bug，也没有新增功能的重构chore: 琐碎的修改，不知道如何分类，不会产生什么影响的改动
breaking change: 破坏性更新，比如修改了底层库名，方法
... 还有更多字段

提交说明的结构如下所示：

<类型>[可选的作用域]: <描述>
[可选的正文]
[可选的脚注]
